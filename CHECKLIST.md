構造チェックリスト（黒景スナップ）

本ファイルは、
判定・評価・説明が実装に入り込む余地がある箇所を特定するための
固定チェックリストである。

以降の変更時は、
差分が本チェックリストのいずれかに触れているかどうかのみを確認する。
	•	改善提案を行わない
	•	判断を行わない
	•	仕様の再解釈を行わない

⸻

Ⅰ. DOM構造軸

判定・評価・説明が混入し得る箇所

1. 見出し要素（h2 / h3）
	•	見出し文言が
	•	因果
	•	強度
	•	優劣
を想起させる語に変更される余地

2. 原文表示ブロック
	•	原文を囲う DOM に
	•	装飾
	•	注釈用ラッパー
	•	比較用コンテナ
が追加される余地

3. 議論断面ブロック（A / B / C）
	•	断面ごとの DOM に
	•	class 名による意味付け
	•	data 属性の増設
が行われる余地
	•	並び順を変更できる構造になっていないか

4. ズレ発生ポイント（ul > li）
	•	<span>（ラベル）と <div>（内容）の関係が
	•	解説構造
	•	注釈構造
に変形される余地

5. 構造タグ表示部
	•	◻ ブレーカーチェック対象 に
	•	補足DOM
	•	アイコン
	•	色分け
が追加される余地

6. 課金導線DOM
	•	payment-text に
	•	意味説明文
	•	誘導文
が追記される余地
	•	<button> 以外の操作DOMが増える余地

⸻

Ⅱ. ルーティング軸

判定・評価・説明が混入し得る箇所

1. / → /display 遷移
	•	遷移条件に
	•	理由分岐
	•	状況説明
が付加される余地

2. /display 直接アクセス無効化
	•	redirect 先で
	•	エラーページ
	•	状態説明ページ
が挟まれる余地

3. リロード時の無効化
	•	リロード検知に伴い
	•	警告
	•	注意表示
が追加される余地

4. 戻る操作時の再評価
	•	履歴遷移時に
	•	条件理由を返すルート
が生まれる余地

⸻

Ⅲ. 状態・フラグ軸

判定・評価・説明が混入し得る箇所

1. showPaymentGate
	•	true / false の意味が
	•	状態説明
	•	ユーザー理解段階
として解釈される余地

2. canDisplayOnce
	•	フラグ名・コメントが
	•	理由
	•	正当性
を含む表現に変更される余地

3. フラグ消失タイミング
	•	フラグ破棄時に
	•	失効理由
	•	注意喚起
を表示する余地

4. フラグの共有範囲
	•	フラグを
	•	props 経由で可視化
	•	context 化
することで意味が発生する余地

⸻

横断チェック（共通）
	•	「なぜ」「どうして」「確認」「理解」などの語を
差し込める余白が残っていないか
	•	判定が UI ではなく
構造として見えてしまう箇所がないか
	•	「失敗」「無効」が
説明なしで成立しているか

⸻

運用固定ルール
	•	本チェックリストは 変更しない
	•	以降の実装変更時は
差分が上記いずれかに触れているかのみを確認する
	•	チェックリストを根拠に
	•	改善
	•	修正
	•	追加
を行わない
